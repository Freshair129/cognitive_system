# PARAMS – Recommended Metadata Profile for Obsidian Notes

นี้คือ **profile** ที่ควรใช้กับทุกไฟล์ Markdown ที่แก้ไขใน Obsidian เพื่อให้ GKS สามารถดึงและแสดง Metadata ได้อย่างครบถ้วน

## Field defaults

| Field | Recommended value | Reason |
|------|-------------------|--------|
| `domain` | `knowledge` | ส่วนใหญ่เป็นข้อมูลความรู้ (ใช้ `policy` หากเป็นนโยบาย) |
| `source_type` | `gks` | โน้ตที่สร้างโดย GKS เอง |
| `created_at` | **Auto‑generated** by script `npm run generate:metadata` (ISO‑8601 +07:00) | ป้องกันการพิมพ์ผิด |
| `summary` | **Optional** – หากไม่มีให้เว้นไว้ (จะไม่แสดงใน tier `SUMMARY`) |
| `tags` | `[]` (empty) – เติมตามโครงการ |
| `attributes.salient` | `""` – ควรใส่ข้อความสั้น ๆ ที่สำคัญ |
| `attributes.trigger` | `""` – เงื่อนไขที่ทำให้โน้ตนี้เกี่ยวข้อง |
| `attributes.hook` | `""` – ลิงก์ย้อนกลับ (ถ้ามี) |

## Example front‑matter (generated by the helper script)

```yaml
---
id: CONCEPT--EXAMPLE-NOTE
title: ตัวอย่างโน้ต
domain: knowledge
source_type: gks
created_at: 2026-05-28T13:45:00.000+07:00
summary: สรุปสั้น ๆ ของโน้ตนี้
tags:
  - onboarding
  - security
attributes:
  salient: "จุดเด่นของโน้ต"
  trigger: "เมื่อผู้ใช้เปิด Obsidian ครั้งแรก"
  hook: "link-to:CONCEPT--RELATED"
---
```

> **Tip:** คุณสามารถใช้สคริปต์ `npm run generate:metadata` (ถ้ามี) เพื่อเติม `created_at` และ `id` อัตโนมัติทุกครั้งที่สร้างไฟล์ใหม่

## How to apply the profile
1. คัดลอกบล็อก YAML ตัวอย่างข้างต้นลงในไฟล์ Markdown ของคุณ
2. ปรับ `domain`, `tags` และ `attributes` ให้ตรงกับเนื้อหา
3. รัน `npm run msp:validate` เพื่อตรวจสอบว่าไฟล์สอดคล้องกับ profile
4. หากมีหลายไฟล์ให้สร้าง **template** (เช่น `templates/obsidian-note.md`) แล้ว copy‑paste ทุกครั้ง

## Validation checklist (for reviewers)
- [ ] `domain` มีค่าเป็นหนึ่งในค่าที่อนุญาต (`knowledge`, `policy`, `operations`)
- [ ] `source_type` เป็น `gks` (หรือ `msp`/`external` หากจำเป็น)
- [ ] `created_at` มีรูปแบบ `YYYY-MM-DDTHH:mm:ss.sss+07:00`
- [ ] `summary` อยู่ในบรรทัดเดียวไม่เกิน 200 ตัวอักษร (แนะนำ)
- [ ] `attributes` มีคีย์ที่กำหนด (`salient`, `trigger`, `hook`) – ค่าที่ว่างก็ได้
