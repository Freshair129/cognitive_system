# Cognitive Model — Shared Brain Philosophy

## 1. The Core Metaphor
The "Brain" is not a part of the software; it is the **soul** of the agent. Just as a human can apply their knowledge to different tools (IDEs, Terminals, CLIs), an AI Agent should have a **Portable Brain** that can be plugged into any compatible "Body" (Code Repository).

## 2. Decentralized Sovereignty
In a multi-agent ecosystem, knowledge must be:
- **Fluid:** Capable of being synchronized across multiple actors.
- **Persistent:** Independent of the transient state of source code branches.
- **Auditable:** Every "thought" (atom update) must be attributable and reversible.

## 3. The Shadow Principle
The "Shadow Repo" acts as the ethereal reflection of the agent's internal state. It is a "shadow" because it follows the agent everywhere, but it is a "repo" because it utilizes the most robust synchronization technology ever built: **Git**.

## 4. Shared vs. Local Memory
Knowledge is shared, but **experience** is local.
- **Shared Memory (The Brain Repo):** Facts, Specs, ADRs, and Protocols.
- **Local Experience (The Brain Index):** Vector embeddings, Graph snapshots, and Session traces.
This distinction allows multiple agents to share the same *Facts* while maintaining their own unique *Perspectives* and *Performance Optimizations*.
